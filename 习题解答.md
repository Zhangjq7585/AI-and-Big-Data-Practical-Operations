# test1-test12习题解答

## test1:电商用户消费数据分析（缺失值和异常值处理）

1.年龄缺失值处理：考虑到年龄与性别可能存在关联，采用按性别分组填充中位数的方式，避免单一值填充导致存在的偏差，  

2.“消费金额”异常值处理：使用IQR(四分位距法识别异常值，计算Q1(25%分位数)和Q3(75%分位数)，将超出[ Q1-1.5*IQR , Q3+1.5*IQR ]范围的值是为异常值处理，
采用Q3+1.5*IQR替换上限异常值、Q1-1.5*IQR替换下限异常值的截断处理，避免删除数据导致样本减少。

## test2:将分类字段转化为数值特征并编码。

1."职业"字段属于无顺序的名义变量，适合使用独热编码(One-Hot Encoding)，避免给不同职业赋予隐含的大小关系。

2.“学历”字段属于有顺序的Ordinal变量(大专<本科<硕士<博士)，适合用标签编码(Label Encoding)，保留学历的等级关系。

## test3:特征选择、任务分类

方法1：基于相关性的特征选择。计算特征之间的皮尔逊相关系数，删除相关系数绝对值大于阈值(如0.8)的高相关特征。
	优点：1.计算效率高，不需要训练模型；2.易于理解和解释；3.可以有效去除冗余特征；4.无监督方法，不依赖标签
	缺点：1.只考虑特征间的线性关系；2.可能删除对模型重要的特征3.需要手动设置相关性阈值。

方法2：基于模型的特征重要性选择。使用随机森林等树模型，根据特征重要性得分筛选Top N特征。
	优点：1.考虑了特征与目标的关系；2.可以捕捉非线性关系；3.提供特征重要性排序；4.通常能保持或提高模型性能
	缺点：1.计算成本较高；2.需要训练模型；3.结果依赖于所选模型；4.可能选择高度相关的特征。

## test4:图像数据转换与降维。

1.数据转换步骤：将28×28的二维图像矩阵展平为一维向量，每张图变为784维特征，形成形状为(10000,784)的特征矩阵。对像素值进行标准化，消除量纲影响；
2.降维方法:使用PCA（主成分分析），这是一种线性降维方法。通过正交变换将相关变量转为不相关的主成分。按方差贡献率选择主成分，保留95%的原始信息，可以有效减少特征维度，去除噪声和冗余信息。

利用Pandas的滚动窗口(rolling)函数，按"日期"排序后，分别计算7天窗口的均值和3天窗口的最大值，构建时间特征。

## test6:构建二分类模型预测客户流失风险。

1.选择的模型：逻辑回归。理由：基础模型，可解释性强；XGBoost。理由：集成模型，对非线性关系拟合能力强。
2.评估指标：二分类任务中，若客户流失为“少数类”（如流失率10%），选择F1 分数（平衡精确率和召回率）和AUC（反映模型区分能力，不受类别不平衡影响），避免仅用准确率导致误判。

## test7:回归模型超参数调优

### 1.网格搜索 (GridSearchCV)

优点：
	系统性地遍历所有参数组合，确保找到网格内的最佳组合
	结果完全可重复
	适用于参数数量较少、取值范围明确的情况
缺点：
	计算成本高，随参数数量指数级增长
	可能无法找到网格范围之外的最佳参数
	对于高维参数空间不实用

### 2.随机搜索 (RandomizedSearchCV)

优点：
	计算效率高，可以在更少的迭代中找到接近最优的参数
	能够探索更广泛的参数空间
	适用于高维参数空间
	可以指定参数的概率分布
缺点：
	可能错过某些重要的参数组合
	结果有一定随机性
	需要合理设置参数分布

### 3.使用建议

​	当参数空间较小且计算资源充足时，优先使用网格搜索
​	当参数空间较大或计算资源有限时，使用随机搜索
​	实践中可以先使用随机搜索缩小参数范围，再使用网格搜索进行精细调优

## test8:解决SVM多分类过拟合的方法

1.增加正则化强度。SVM中通过C参数控制正则化（C越小，正则化越强，避免过拟合）；
2.使用核函数正则化。对RBF核，通过gamma参数控制核函数复杂度(gamma越小，核函数越简单，减少过拟合)；
3.数据增强。增加训练样本数量（如对图像数据旋转、裁剪、对结构化数据添加轻微噪声）；
4.特征选择。删除冗余或噪声特征、降低模型输入维度。

## test9:处理信用卡欺诈检测数据的类别不平衡的方法

1.调整样本权重。通过class_weight(LightGBM为scale_pos_weight)设置正负样本权重，权重比例为“负样本数/正样本数”；
2.过采样（SMOTE）。对少数类（欺诈样本）进行合成采样，增加少数类样本数量；
3.欠采样。对多数类（正常样本）随机删除部分样本，使两类样本数量接近；
4.集成方法（EasyEnsemble）.将多数类拆分为多个子集，与少数类组成多个训练集，训练多个模型后融合。

## test10:模型融合：投票法与堆叠法实现

​	投票法和堆叠法的区别:

| 特性       | 投票法 (Voting)                          | 堆叠法 (Stacking)                      |
| ---------- | ---------------------------------------- | -------------------------------------- |
| 基本原理   | 并行组合多个模型的预测结果               | 分层结构，使用元模型学习如何组合基模型 |
| 训练方式   | 所有基模型独立训练                       | 两阶段训练：基模型 → 元模型            |
| 信息利用   | 仅使用最终预测（硬投票）或概率（软投票） | 使用基模型的预测/概率作为新特征        |
| 复杂度     | 简单、计算量小                           | 较复杂，需要训练元模型                 |
| 过拟合风险 | 较低                                     | 较高，需要交叉验证防止过拟合           |
| 可解释性   | 较好（直观的投票机制）                   | 较差（黑箱元模型）                     |
| 数据使用   | 直接使用原始数据                         | 通常使用交叉验证生成元特征             |
| 灵活性     | 固定组合规则                             | 元模型可以学习最优组合方式             |
